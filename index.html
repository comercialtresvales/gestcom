🔥 Firebase Realtime Database
100% Gratuito
Firebase Console → Realtime Database → copie a URL
A API Key só é necessária se as regras do Firebase exigirem autenticação. Se você configurou as regras como ".read":true, ".write":true, deixe este campo vazio.
📋 Como configurar (3 minutos)
1Acesse console.firebase.google.com → crie um projeto
2Vá em Build → Realtime Database → Criar banco
3Copie a URL: https://SEU-PROJETO-default-rtdb.firebaseio.com
4Vá em Regras e cole exatamente isso:
{
  "rules": {
    ".read": true,
    ".write": true
  }
}
5Cole a URL no campo ao lado → clique Conectar → API Key: deixe vazio
6Clique 📤 Enviar dados para nuvem → pronto! ✅
⚠️ Regras abertas (.read/.write: true) são suficientes para uso interno. Se quiser restringir acesso externo, fale com seu desenvolvedor para configurar autenticação.
⚙️ Comportamento da Sincronização
✅ Ativado — cada ação salva automaticamente em até 1,5 s
✅ Ativado — atualiza dados da equipe automaticamente
💡 Auto-salvar ativado? Cada vez que você cadastra, edita ou exclui qualquer dado, o sistema salva automaticamente na nuvem em 1,5 segundo — você verá o indicador ● Salvando... piscar no topo.
📥 Atualização automática? A cada 5 segundos o sistema verifica se alguém da equipe salvou algo novo e atualiza a tela.
📤 Enviar para nuvem sobrescreve a nuvem com seus dados locais. Baixar da nuvem traz os dados mais recentes da equipe.
